# VBT Cost Calculator – Implementation Assumptions

## Architecture

1. **PDF Generation**: Using `@react-pdf/renderer` instead of Playwright/Puppeteer. This works server-side in Next.js without requiring a browser binary, making it suitable for Vercel deployments. The tradeoff is slightly less HTML-fidelity, but the custom PDF layout is production-quality.

2. **Authentication**: NextAuth v4 with JWT strategy + credentials provider. Users sign up and await SUPERADMIN approval before access is granted. No OAuth providers are configured but can be added.

3. **API Architecture**: Next.js App Router API routes (not tRPC) for simplicity and better alignment with Vercel edge compatibility. All mutations use server-side route handlers with Zod validation.

4. **Multi-tenancy**: Single org ("Vision Latam") by default. The schema supports multiple orgs via orgId foreign keys on all entities. Adding new orgs requires a seed extension or admin UI (not implemented in v1.0).

5. **Database**: Neon Postgres via Prisma. All IDs use cuid() for better readability vs UUID. The schema uses `@map` directives for snake_case table names.

## Business Logic

6. **S150 system**: The 6in category maps to S150 (6in ≈ 152.4mm, rounded to 150mm for the system code). The UI and reports refer to it as "S150 (6in)".

7. **S200 system**: The 8in category maps to S200 (8in ≈ 203.2mm, rounded to 200mm).

8. **Min Run Alert**: When a piece's total linear feet in a CSV import is below the org's `minRunFt` (default 5000 ft), an alert is shown and a markup percentage can be applied. The markup applies to that piece's line total only.

9. **Commission**: Commission can be both percent AND fixed USD simultaneously. The formula is: `commission = factoryCost * pct/100 + fixed`. This covers the common pattern of percent + fixed fees.

10. **Tax BASE_IMPONIBLE (Argentina)**: BASE_IMPONIBLE is computed as `CIF + duty + statistic`, where duty and statistic are lines that have `base = CIF` and appear before the BASE_IMPONIBLE lines (ordered by `order` field). The system accumulates any CIF-based lines labeled "duty" or "statistic" (case-insensitive) to build this base.

11. **Freight**: Entered per-container. `totalFreight = freightPerContainer * numContainers`. Manual override is supported at quote time.

12. **Container count**: `numContainers = ceil(totalKits / kitsPerContainer)`. If kitsPerContainer is 0, numContainers defaults to 1. This avoids division by zero for tax calculations.

13. **Inventory reserve**: Reserves are created at quote creation if `reserveStock = true`. They are released when a quote is cancelled/archived. Inventory can go into deficit (production_needed) but doesn't block quoting.

14. **Quote snapshot**: The full `buildQuoteSnapshot()` output is stored in `Quote.snapshot (JSON)`. This freezes the calculation state at quote time, meaning future rate/tax changes do not affect past quotes.

## Data Model

15. **PieceAlias uniqueness**: The constraint is `(aliasNormalized, pieceId)` to allow the same normalized name to map to the same piece from multiple sources without error.

16. **PieceCost ID**: The seed and import use deterministic IDs like `seed-{pieceId}` and `import-{pieceId}` to enable upserts. A new cost entry can be created by using a random ID (the default `cuid()`).

17. **System thickness**: S80=80mm, S150=150mm, S200=200mm. These are named sizes, not exact thickness values for concrete calculation.

18. **RevitImportLine.matchMethod**: Possible values are `EXACT_CODE`, `ALIAS`, `CANONICAL`, `MANUAL`, `IGNORED` (not an enum in DB to allow flexibility).

## Email

19. **Resend integration**: Email sending uses Resend. PDF is generated by calling the internal `/api/quotes/[id]/pdf` endpoint from the email handler with the same session cookie. If PDF generation fails, the email is sent without an attachment.

20. **Email from address**: Defaults to `quotes@visionbuildingtechs.com`. Must be a verified domain in Resend.

## CSV Import

21. **Header mapping**: Headers are matched with case-insensitive partial matching against synonym lists (see `csv-parser.ts`). Common Revit export headers like "Count", "Type", "Height", "Familia" are all supported.

22. **Unmatched pieces**: The UI requires all lines to be either mapped or explicitly ignored before proceeding. Ignored lines are excluded from all calculations.

23. **CSV height units**: Assumed to be in millimeters (mm). If your Revit export uses different units, pre-convert before uploading or contact the admin.

## UI/UX

24. **shadcn/ui**: Standard shadcn/ui components are used. The `components/ui/` directory must be populated via the shadcn CLI or the provided implementations.

25. **Responsive design**: The app is designed for desktop use (internal tool). Basic responsive layout is included but not fully optimized for mobile.

26. **Pagination**: Project and quote lists use server-side pagination with defaults of 50 items per page.

## Deployment

27. **Vercel**: The app is designed for Vercel deployment. The `serverComponentsExternalPackages` config excludes `@prisma/client` and `bcryptjs` from edge bundling.

28. **Environment variables**: Copy `.env.example` to `.env` and fill in all values before running. Required: `DATABASE_URL`, `NEXTAUTH_SECRET`, `NEXTAUTH_URL`.

29. **PNPM workspaces**: Use `pnpm install` from the root to install all dependencies. Use `pnpm db:migrate` to run Prisma migrations, `pnpm db:seed` to seed.

## Known Limitations / Future Enhancements

- No bulk quote editing
- No quote versioning (only snapshot at send time)
- No multi-currency support (everything is USD)
- Freight profile selection at quote time is manual; no auto-suggestion
- No Revit plugin integration (CSV is exported manually)
- No dashboard analytics/charts (planned for v1.1)
- No email templates system (HTML is inline in the route)
- Steel and concrete estimates are informational only and not included in cost
